## 🔧 Comandos útiles

### Build local
```bash
bun run build
# Debe terminar con "✓ built in Xs"
```

### Verificar git status
```bash
git status
git log --oneline -10
```

### Coordinación con Lovable
1. Hacer cambios localmente
2. Push a main
3. Pegale a Lovable: `Pull main commit <SHA>. Hard refresh.`
4. Lovable verifica y deploya

### Edge functions (Lovable owns)
NO podemos hacer apply directo. Cambios edge fn requieren:
1. Editar archivo en `supabase/functions/<name>/index.ts`
2. Commit + push
3. Pegale a Lovable: `Pull main + deploy edge function <name>`

### Migrations (Lovable owns)
NO podemos hacer apply directo. Cambios migration requieren:
1. Crear archivo `supabase/migrations/PENDING_<name>.sql` (header dice PENDIENTE)
2. Commit + push
3. Pegale a Lovable: `Pull main + renombrar PENDING_<name>.sql con timestamp real + aplicar migration`
4. Lovable confirma apply
5. Editar header para decir "✅ APLICADA" (no dejar PENDING)
